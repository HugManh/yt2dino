# YT2Dino

A modern, fast, and lightweight YouTube Search & Downloader Desktop Application built with Electron, React, and Vite. It allows users to easily search for YouTube videos, view video details, and download them in high quality (Video) or automatically-tagged Audio (MP3).

## Key Features

- **Built-in Search Engine:** Search for YouTube videos directly within the app using `youtube-sr` with infinite scrolling support.
- **Video & Audio Downloads:** Download directly to MP4 or extract audio to MP3 using `yt-dlp`.
- **Automatic ID3 Tagging:** MP3 downloads are automatically tagged with Title, Artist, Year, and Thumbnail cover art using `node-id3`.
- **Modern UI/UX:** A clean, borderless window design with custom titlebar controls, built with React.
- **Cross-platform:** Powered by Electron, ensuring compatibility across macOS, Windows, and Linux.

## Tech Stack

- **Language:** TypeScript, HTML, CSS
- **Framework:** Electron, React 18
- **Build Tool:** Vite, electron-vite
- **Downloading & Conversion:** `yt-dlp-wrap`, `ffmpeg-static`
- **Metadata Handling:** `node-id3`
- **Scraping:** `youtube-sr`

## Prerequisites

- [Node.js](https://nodejs.org/) 20 or higher
- npm or yarn

## Getting Started

### 1. Clone the Repository

```bash
git clone https://github.com/HugManh/yt2dino.git
cd yt2dino
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Start Development Server

This will start both the Vite dev server for the React frontend and the Electron app:

```bash
npm run dev
```

### 4. Build for Production

To compile the application and build the executables for your platform:

```bash
npm run build
```

## Architecture

### Directory Structure

```
├── .vscode/               # VSCode extension configurations
├── out/                   # Compiled outputs (generated by Vite)
│   ├── main/              # Compiled Electron main process
│   ├── preload/           # Compiled preload scripts
│   └── renderer/          # Compiled frontend static files
├── src/                   # Source files
│   ├── main/              # Electron main process code (Node.js)
│   │   ├── index.ts       # Main entry point & window management
│   │   └── ipc-handlers.ts# IPC communication handlers
│   ├── preload/           # Preload scripts (Context Bridge)
│   │   └── index.ts       # Exposing secure APIs to the renderer
│   └── renderer/          # React frontend code
│       ├── index.html     # Main HTML template
│       └── src/
│           ├── components/# Reusable React components (VideoCard, DownloadModal)
│           ├── App.tsx    # Main React application component
│           ├── main.tsx   # React DOM rendering entry
│           └── env.d.ts   # TypeScript environment declarations
├── tests/                 # Unit tests and E2E scripts
├── electron-builder.yml   # Configuration for electron-builder
├── electron.vite.config.ts# Vite configuration for Electron
├── package.json           # Project dependencies and script commands
└── tsconfig.json          # TypeScript configurations
```

### Data Flow & IPC Lifecycle

1. **Frontend Action:** User interacts with the React UI (e.g., searching or initiating a download).
2. **Context Bridge:** The `window.api` (defined in `preload/index.ts`) is invoked securely.
3. **Main Process:** `ipcMain` handlers in `main/ipc-handlers.ts` receive the event.
   - For searches, `youtube-sr` fetches metadata.
   - For downloads, `yt-dlp-wrap` initiates the download, passing progress updates back to the renderer via `webContents.send`.
4. **UI Update:** React state updates trigger re-renders to show real-time progress.

## Troubleshooting

### Failed to Load `yt-dlp`
**Error:** Dependency check fails on app startup.
**Solution:** Ensure you have a stable interent connection on the first launch so `yt-dlp-wrap` can download the necessary `yt-dlp` executable binary for your system.

### Audio Conversion Fails
**Error:** Stuck at extraction or ffmpeg not found.
**Solution:** The app uses `ffmpeg-static`. In rare cases on some OS structures, you might need to ensure execution permissions are granted to the downloaded binary inside `node_modules/ffmpeg-static/`.
